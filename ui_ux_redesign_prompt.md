# Medicinder UI/UX Redesign Prompt

**Role & Goal:**
You are an expert UI/UX Designer. Your task is to redesign the UI and user experience for "Medicinder", a comprehensive medication tracking and reminder mobile application. 

The goal is to elevate the visual aesthetic, making it feel premium, modern, intuitive, and highly engaging. You are generating UI design mockups/concepts (similar to Figma screens). **You do not need to write code or worry about the technical implementation.** Focus entirely on the visual layout, user journey, and aesthetic excellence.

---

## 1. Application Overview
Medicinder helps users keep track of their daily medications, reminds them when it's time to take them, and logs their history. It must feel like a reliable, health-focused app, but with a sleek, modern visual appeal rather than a sterile, clinical look.

## 2. Core UI Screens & Flows Needed
Please design UI screens for the following areas, ensuring all necessary UI elements are present:

### A. Home Dashboard (The main screen)
- **Top Area:** Needs a clean app header/title, a "Statistics" icon button, and a "Settings" icon button.
- **Sync/Status Banner:** A small unintrusive banner indicating if the app is syncing to the cloud or offline.
- **Medication List (Today's Doses):** 
  - A visually appealing list or grid of medications due today.
  - Distinguish clearly between doses that are:
    1. **Completed (Taken)**
    2. **Due Now / Pending**
    3. **Upcoming**
- **Action Elements:** There must be a satisfying, clear way for the user to check off a medication as "Taken".
- **Floating Action Button (FAB):** A prominent button to add a new medication.
- **Bottom Navigation:** Two main tabs: "Home" (Current Doses) and "History" (Last Taken).

### B. Add Medication Screen (Wizard / Form)
- An intuitive input flow to add a new medication. It needs fields/selections for:
  - Medication Name
  - Usage Instructions & Dosage amount
  - Form Factor: Visual toggles for "Pill" vs. "Syrup"
  - Timing: Toggles between specifying an exact Time (e.g., 08:00 AM) or a Context (e.g., "Before Breakfast", "After Lunch").
  - Duration: Number of days, or a toggle for "Repeat Forever".

### C. History / Last Taken Screen
- A timeline or list layout showing doses that were recently consumed. Each entry should show the time it was taken and the medication name.

### D. Statistics Screen
- Visual data representations (charts, progress rings, or graphs) showing user compliance, days left in treatment courses, and adherence rates.

### E. Settings Screen
- Clean layout for managing user profile, toggling languages (English / Arabic), and cloud sync status.

## 3. UI/UX Aesthetic Directives
1. **Modern Aesthetic:** Avoid generic, flat boxes. Use modern mobile patterns. Think rounded corners, soft drop-shadows, varied typography weights, and potentially glassmorphism or minimalist surface layers.
2. **Color Palette:** The app currently uses a teal default (`#71C0B2`), but feel free to upgrade or modernize this. It should feel calm, medical, yet highly polished and vibrant. 
3. **Empty States:** Design beautiful, encouraging empty states for when the user has no medications added yet.
4. **Typography & Hierarchy:** Ensure excellent readability. Important information (like what medicine to take right now) should draw the most focus.
